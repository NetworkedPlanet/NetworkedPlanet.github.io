workerThreads

 Top  Previous  Next

This element allows the configuration of the number of worker threads that the Web3 Job Processing Service uses. Having more worker threads enables more jobs to be processed in parallel but puts more strain on the resources of the machine that hosts the service. The element has a single optional attribute named max. The value of this attribute specifies the number of worker threads managed by this service. If this attribute is not present, the default value is 5.

 

This element is a child of the jobService element.
